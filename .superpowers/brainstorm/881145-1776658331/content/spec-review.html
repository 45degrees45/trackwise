TrackWise Phase 5 — Design Spec

Task Completion Timing & Dashboard  |  2026-04-20

Problem

  1. Tasks completed via Chrome OS Flex Focus mode are not appearing in the MASTER sheet. Root cause: Phase 2 sync.gs only reads the default Google Tasks list — Focus mode uses a separate list.
  2. No time-based analytics — no way to see when tasks are completed (hour of day, day of week, month, year).

Architecture

Google Tasks (ALL lists, including Chrome OS Focus)
        ↓  every 10 min (Apps Script time trigger)
  sync_timelog.gs
        ↓
  TIME_LOG sheet  ←── source of truth for analytics
        ↓  COUNTIFS / FILTER formulas
  DASHBOARD sheet

Zero cost — runs on Google's servers. Ubuntu machine does not need to be on.

sync_timelog.gs — Behaviour

  • Fetches all task lists via Tasks.Tasklists.list() — catches Focus mode tasks
  • Fetches completions since last sync cursor (stored in PropertiesService)
  • Appends rows to TIME_LOG with full timestamp breakdown
  • Idempotent — checks Task Name + Completed Date before inserting
  • Registers 10-min time trigger on first run via setupTimeTrigger()

DASHBOARD — Sections

  • Today Stats — tasks today, this week, peak hour, day streak
  • Weekly Hour Heatmap — 6am–7pm, green scale by count
  • Monthly Day Heatmap — calendar grid, current month
  • Yearly Month Heatmap — Jan–Dec bar, current month highlighted
  • Recent Completions — last 10 tasks with timestamps

Deployment Steps

  1. Open TrackWise Google Sheet → Extensions → Apps Script
  2. Create new file sync_timelog → paste script
  3. Run setupTimeTrigger() once — creates 10-min trigger + TIME_LOG sheet
  4. Enable Tasks API: Services → Tasks API v1
  5. Run syncCompletionTimes() manually once to backfill
  6. Create DASHBOARD sheet, paste formula blocks from DEPLOY.md
  7. Verify Chrome OS Focus tasks appear in TIME_LOG

Success Criteria

  • Chrome OS Flex Focus completions appear in TIME_LOG within 10 min
  • DASHBOARD shows correct counts for today / this week
  • All 4 heatmaps reflect real data
  • No duplicate rows on repeated sync
  • Runs without Ubuntu machine being on
