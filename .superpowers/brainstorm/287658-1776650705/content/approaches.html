TrackWise — Task Completion Timing

When you strike off a task in Google Tasks → log time → dashboard in Google Sheets

A

Fix + Extend Phase 2 (Apps Script only)

Fix the existing sync.gs to read from ALL Google Tasks lists (including Chrome OS Focus tasks). Add a new TIME_LOG sheet that records task name + exact completion time. Build a dashboard tab with time-of-day charts.

Pros

  • Zero new infrastructure
  • Runs on Google's servers — Ubuntu machine doesn't need to be on
  • Free forever
  • Everything in one Google Sheet

Cons

  • Polling every 10 min — not real-time
  • Apps Script has execution limits
B

Dedicated TIME_LOG Sheet + Charts Dashboard

Keep MASTER as-is. Add a separate TIME_LOG sheet just for timing data. A new Apps Script writes: Task Name, List, Completed Date, Completed Time, Hour of Day, Day of Week. Dashboard tab shows heatmaps and completion trends.

Pros

  • Clean separation — MASTER for tasks, TIME_LOG for analytics
  • Easier to build charts
  • Still zero cost

Cons

  • Two sheets to maintain
C

Google Apps Script + Looker Studio Dashboard

Apps Script logs completion times to Sheets. Connect Looker Studio (free, by Google) to the sheet for richer charts — hourly heatmaps, weekly trends, streak visualizations — without building charts manually in Sheets.

Pros

  • Professional dashboard visuals
  • Zero cost (Looker Studio is free)
  • Live updates

Cons

  • Extra setup step (Looker Studio connection)
  • Dashboard is a separate URL from the Sheet

My recommendation: Option B — clean separation between task tracking and time analytics, zero cost, everything in Google Sheets. Chrome OS Focus tasks get fixed as part of the sync rewrite.
